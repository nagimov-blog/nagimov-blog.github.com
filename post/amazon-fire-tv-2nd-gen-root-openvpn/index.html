Amazon Fire TV 2nd Gen - Root & OpenVPN

    
2016-03-30

My notes after spending half a day playing with my Fire TV.

While Amazon Fire TV seems to be the best htpc solution for me, it still requires rooting and a bit of patching on its way to a decent media center. I’m also used to US streaming services (Pandora, Netflix, Hulu, etc.) that are only available through VPN, if you are outside of US. Rooting and setting up Kodi + OpenVPN is a way to go.

Rooting is simple and straightforward, I followed this guide and everything went smooth.

Problems started with OpenVPN. Even though there are few tutorials out there (here and there), I wasn’t able to install OpenVPN using the installer apk.

Another tutorial suggests to do installer’s jobs manually (i.e. unpack OpenVPN binary to /system/xbin/). I’ve done that and got only position independent executables (PIE) are supported error when trying to start OpenVPN from adb console. Apparently, there is a known issue with position independent executables (whatever that means). The first solution is to find the OpenVPN binary compiled with PIE support (or compile it from sources adding "-fPIE -pie" linker options), that could take awhile. Another way to deal with the PIE issue is to patch the linker binary located in /system/bin/ directory. I’ve tried a bunch of different versions of the linker file, sometimes getting Android to behave weirdly (e.g. randomly give application storage space is full errors) or even screwing my Fire TV to the boot-loop. This is the version of the patched linker that worked great with my Fire TV.

Here’s how to replace linker file in /system/bin/ directory using adb:

adb push linker /sdcard/
adb shell
su
mount -o remount rw /system
mv /system/bin/linker /system/bin/linker.bak
cp /sdcard/linker /system/bin/
chmod 755 /system/bin/linker 
mount -o remount ro /system
rm /sdcard/linker

Happy Kodiing!
